CSS Positioning - setting the scene

  • In the past, web sites used HTML tables for layout and Cascading Style Sheets (CSS) for the text formatting.
  • There are good reasons why tables shouldn't be used for layout:
    • Strictly speaking HTML tables should be used for tabular data only
    • Doesn't separate presentation from content (you need to check your tables linearise properly)
    • Tabular layouts are complicated to maintain
    • Larger file size for your HTML file
    • It's old fashioned

CSS Positioning - setting the scene

  • In this session we'll look at the basic concepts underlying CSS positioning (CSS-P) and some of the code that makes it possible.
  • This session will not give you definitive answers/solutions to producing particular page layouts. It will provide code examples and explain how page elements 'interact'.
  • CSS-P can be complex, confusing and frustrating to start with. Don't lose heart but keep practising.

The Box Model

Understanding CSS positioning relies on a understanding the Box Model.

Everything on an HTML page fits into a box.

All boxes have: Margin, Border, Padding, Content

There are two types of box:

  • Block - Form new blocks of content
    e.g. h1, p and div tags.
  • Inline - Do not form new blocks but lines of content
    e.g. em, strong and span tags.

Div and Span reminder

One of the key building blocks of CSS-P is being able to group elements together using <div> and <span> tags.

  • div - block level container (short for division)
  • span - inline container

<div> and <span> have no inherent properties/styles of their own so can be used to apply styles to the content within them.

CSS positioning schemes

Most of the examples we will look at demonstrate how elements can be arranged using static positioning and floats. This is arguably the commonest form of CSS positioning.

However, there are four positioning schemes (elements can only have one):

  • position: static; - normal page layout rules (block boxes stack vertically). Usually combined with floats. As it's the default you very rarely set it.
  • position: relative; - positioned relative to where the box would normally appear. A gap is left where the box would be.
  • position: absolute; - positioned relative to a 'positioned' parent box.
  • position: fixed; - the same as absolute except positioned with respect to the browser window.
  • position: sticky; - toggles between relative and fixed. Element is positioned relative until a threshold is met and then uses fixed positioning.

relative, absolute, fixed and sticky are used less often than float because they create designs that don't scale well.

CSS Grid Layout (Grid) is an exciting new layout system in CSS but doesn't have complete support yet.

CSS positioning offsets

When using relative, absolute or fixed, boxes can have four offsets:

  • top
  • right
  • bottom
  • left

However, it doesn't make sense to set both top and bottom or both left and right at the same time. You can set top and left etc though.

So, to move a box you might do something like:

#navigation {
position: absolute;
top: 150px;
left: 200px;
}

Browser support

The level of support for CSS positioning in browsers varies.

  • Current versions of Chrome, Firefox, Safari, Edge are all very good.
  • IE7 is OK, IE8 onwards are good.

The secret to CSS Positioning is testing.

Browser support

Test your Web site in as many Web browsers as possible before launch.

In addition you should also be aware of browser "Quirks mode"

When using CSS Positioning it's really important that you create valid HTML and CSS.

CSS media types

CSS isn't only used to control how a Web page appears on a PC screen. Different stylesheets can, for example, be used to make sure your pages print correctly.

Different types of stylesheet are identified using the 'media' attribute. e.g.:

<link rel="stylesheet" type="text/css" media="screen" href="mystyles.css">
<link rel="stylesheet" type="text/css" media="print" href="print.css">

Further info: CSS Media types and Media Queries (more on media queries in the Web project module).

Development approach

  • Develop in a browser with good support for Web standards:
    • Firefox, Chrome, Safari
    • Don't develop in any version of IE
  • Then test in IE/Edge
  • Edit your CSS and retest
  • Avoid mixing positioning types as much as possible. I almost always work with static (browser default) positioned elements and then use floats. I rarely use other positioning types and avoid 'fixed' as the resulting site is usually inflexible and degrades badly on different screen sizes.
  • Avoid the temptation to set a height on any element that contains content that will vary in length - let the browser do the work.
