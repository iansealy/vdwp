Types of Layout

Progression of layout development:

  • Fixed width (px)
  • Fluid/liquid width (%)
  • Elastic width (em)
  • Responsive

Basic HTML

  • Menus from Listamatic

Fixed Width

  • Size in px
  • Easiest to develop
  • Can make sure content is optimal reading width (unless user changes font size)
  • Example

Fluid or Liquid Width

  • Size in %
  • Easy to develop
  • Example
  • User has control of optimal reading width (but will they)
  • Example

Elastic Width

  • Size in em
  • Harder to develop
  • Width adapts to font size
  • But modern browsers support page zoom
  • Example

Aside: CSS3

  • Modular extension of CSS2.1
  • Some modules are finished and have wide support; others do not
  • CSS Validation Service defaults to CSS3
  • e.g. border-radius and selectors
  • CSS3 also introduced media queries - way of adapting CSS based on user agent characteristics (e.g. screen size)

Mobile Layout / Responsive Design

  • Could have separate site on subdomain (e.g. m.bbkweb.org)
  • But maintenance becomes harder, e.g. what about tablets?
  • Better to have design adapt to screen size
  • Responsive Web Design - a combination of fluid design (%), flexible images (width: 100%) and media queries
  • Example
  • Better would be mobile-first design using min-width
