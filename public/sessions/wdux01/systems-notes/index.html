File Extensions

By default Windows hides file extensions, but this is annoying for Web development. For example, football.html and football.htm are different files, but will appear to be the same in Windows. So you need to always show all file extensions. The exact method depends on the version of Windows you are using, but is usually either done by opening Windows Explorer and then choosing Tools --> Folder Options --> View tab and then unchecking "Hide extensions for known file types" or by opening Windows Explorer and then choosing Organize --> Folder and search options --> View tab and then unchecking "Hide extensions for known file types". You'll want to make this change on your home computer as well as at Birkbeck.

Web Space

Each student has their own Web site on a DreamHost server. The URLs are all of the form http://username.wdux1718.bbkweb.org/. So, for example, the URL of my Web space is:

  • http://iansealy.wdux1718.bbkweb.org/

A list of all the URLs is available from:

  • http://wdux1718.bbkweb.org/students/

Your username is the same for Birkbeck and for DreamHost, but the passwords are different. You can manually keep them in sync if you wish. Changing your Birkbeck password will not change your DreamHost password (and vice versa). See above for changing your Birkbeck password. Changing your DreamHost password is more complicated and involves logging into a remote DreamHost server. On Windows, this can be done using a program called PuTTY that's installed on the Birkbeck machines and can also be downloaded for free from:

  • http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html

Run PuTTY and then in the Host Name box type bbkweb.org and press the Open button. When the "login as:" prompt appears, type your username and press Enter. Then enter your password and press Enter. (Note that there won't be any indication that you're typing your password in. You just have to assume you are!) You should get logged into the Birkbeck server. Once you've logged in type passwd (in lower case) and then, when prompted, enter your existing password and press Enter. Then enter your new password (twice). If the system doesn't accept your new password then you'll need to run passwd again and retry. If the system accepts your new password then type exit and press Enter.

If you're using a Mac then run Terminal and type ssh username@bbkweb.org (changing username to your DreamHost username). When asked for your password, type it in and press Enter. You should get logged into the Birkbeck server. Once you've logged in type passwd (in lower case) and then, when prompted, enter your existing password and press Enter. Then enter your new password (twice). If the system doesn't accept your new password then you'll need to run passwd again and retry. If the system accepts your new password then type exit and press Enter.

FTP

In order to put files on your Web site you will need to upload them to the DreamHost server. This is possible via FTP. FTP stands for File Transfer Protocol and simply allows you to transfer files from one computer to another.

There are scores of different FTP programs available, but they all follow the same basic scenario:

  • Specify remote computer to connect to
  • Give username and password
  • Either specify files to GET from the remote computer or PUT files from the local computer onto the remote computer

For accessing your DreamHost Web space, the computer you want to connect to is:

  • bbkweb.org

You'll then need to login using your DreamHost username and password. Once you've been authenticated you'll be able to either GET files from the remote computer (the DreamHost Web server) or PUT files from your computer onto the DreamHost Web server.

FTP Clients

In order to access your Web space via FTP, you'll need an FTP client. We recommend you install FileZilla, which should also be on the Birkbeck computers. The current version, for Windows and Macs, can be downloaded from:

  • http://filezilla-project.org/download.php?type=client

After you've downloaded FileZilla, you can install it by simply double clicking on it. On the first screen is a License Agreement, which you should agree to. The default options on the next few screens are all OK and you can just accept them.

Once FileZilla is installed, you can run FileZilla by choosing it from the Start menu. The FileZilla window is split into two main panes. The left-hand pane shows the files on your local computer. The right-hand pane shows the files in a directory on the remote computer. Since you haven't yet connected to a remote computer, the right-hand pane will be blank. Above these two main panes is a pane showing connection information and below is a pane showing the details of current transfers. Both of these panes will currently be blank. The simplest way to use FileZilla is to fill in three of the boxes at the top of the screen. In the "Host" box you should enter:

  • bbkweb.org

In the "User" box you should enter your DreamHost username. If you forget your username then a list is available on the course Web site at:

  • http://wdux1718.bbkweb.org/students/

In the "Password" box you should enter your DreamHost password. Finally you click on the "Quickconnect" button. You should then connect to the DreamHost server and the files in your Web space will be visible in the right-hand pane of the FileZilla window.

If you want to transfer a file from your computer to the Web server then you should navigate to that file in the left-hand pane and then drag and drop it into the right-hand pane.

If you want to transfer a file from the Web server to your PC so that you can work on it then you follow the same procedure but in reverse. You should navigate to the file in the right-hand pane and then drag and drop it to the left-hand pane.

Web Site Structure

Any files that I upload to the htdocs directory of my Web space will be visible on the Web. For example, if I FTP a file called ian.html to the htdocs directory then I will be able to view this file on the Web by appending the name of the file to my base URL and going to:

  • http://iansealy.wdux1718.bbkweb.org/ian.html

If I create a subdirectory in htdocs called birkbeck and put a file in that directory called kieren.html then that file will be visible on the Web at:

  • http://iansealy.wdux1718.bbkweb.org/birkbeck/kieren.html

That is, I have appended the directory (birkbeck) and the name of the file (kieren.html) to my base URL (http://iansealy.wdux1718.bbkweb.org/). Each component of the URL is separated with a forward slash.

If I create a subdirectory of birkbeck called college then the URL of that directory will be:

  • http://iansealy.wdux1718.bbkweb.org/birkbeck/college/

Index Documents

If I were to put a file called index.html in the birkbeck/college directory then instead of seeing a default directory listing when I went to:

  • http://iansealy.wdux1718.bbkweb.org/birkbeck/college/

I would instead see the contents of the index.html document.

The index.html file is just an ordinary file and, in fact, it can be viewed normally. The following two URLs would produce exactly the same page:

  • http://iansealy.wdux1718.bbkweb.org/birkbeck/college/
  • http://iansealy.wdux1718.bbkweb.org/birkbeck/college/index.html

The DreamHost Web server is configured so that if no index document is present then a directory listing is shown instead. This doesn't have to be the case and Web servers can be configured so that an error message is shown instead. An example of this can be seen at:

  • http://www.amentsoc.org/ssi/

An index document doesn't have to be called index.html, although this is the most common. Other examples are index.htm, welcome.html, home.html and Default.htm. The latter is commonly used on Windows-based Web servers.

For another explanation of index documents, have a look at:

  • http://webdesign.about.com/od/beginningtutorials/f/index_html.htm
