Domain Registration And Web Hosting

Ian Sealy

Registering Domain Names

  • Typing address into Web browser is not enough to check if domain is already registered
  • Domain name may be registered, but not associated with a Web server (e.g. just used for email or not used at all)
  • Need to use a WHOIS tool which looks up records in the registry database
  • Rumours abound of the dangers of looking up a domain using some registrars
  • If paranoid, use registry for particular top level domain (TLD)
  • For example, for .uk use Nominet
  • To find registries in other countries, see IANA

Registering Domain Names

  • (Registrars register domains with registries on behalf of their clients, but registries are also registrars for their own registry)
  • Don't have to register with the registries
  • Should choose registrar that combines good price with good service
  • I currently use Joker for .com/.net/.org domains and Namecheap for .uk domains.
  • Hosting providers will often offer free domain registration (e.g. DreamHost)

Backordering Domain Names

  • If a domain you want isn't available then can backorder it
  • Try to buy domain if it becomes available
  • Fee may or may not be refundable if not successful
  • Sometimes ends up as an auction
  • e.g. SnapNames and Sedo

Top-Level Domains

  • Up until 2000, were seven generic TLDs (of which only first three are available to people worldwide):
    • .com
    • .org (may in future be reserved for non-profit making organisations)
    • .net
    • .gov (reserved for US government)
    • .edu (reserved for US educational institutions)
    • .mil (reserved for US military)
    • .int (reserved for organisations established by international treaties)
  • Plus country-code TLDs (e.g. .uk or .ly)

Top-Level Domains

  • Seven new TLDs were accepted in 2000:
    • .aero (for air transport industry)
    • .biz (for businesses)
    • .coop (for cooperatives)
    • .info (for anything)
    • .museum (for museums)
    • .name (for individuals)
    • .pro (for accountants, lawyers and doctors)
  • Others have been created since:
    • .jobs (for recruitment)
    • .mobi (for mobile products and services)
    • .travel (for travel and tourism businesses)
    • .tel
    • .xxx
  • Wikipedia list of TLDs
  • Many more TLDs created in last few years

Hosting

Once domain has been registered, it must be hosted. In general are three options:

  • Host site on own server on own network
    • Must support server and support network
    • Only suitable for large organisations who can take advantage of similar economies of scale to hosting providers

Hosting

  • Colocate own server with hosting provider
    • Computer is much lower cost than network
    • Only suitable if need to own box (e.g. for legal reasons) or have very specific hardware requirements
    • Might be done if need large number of servers
    • Still need technical support to administrate server (e.g. install security patches)

Hosting

  • Host site on a hosting provider's server
    • Cheapest option
    • Hosting provider (probably) takes care of technical support (backups, upgrades, redundancy, monitoring, security, etc...) leaving you to concentrate on Web development
    • Have someone to blame if SLA not met!
  • Types:
    • Dedicated server
    • Virtual private server
    • Shared hosting
    • Cloud

Hosting

  • Price hosting provider charges for hosting varies according to features needed
  • Common features that increase cost are:
    • Increasing amounts of bandwidth
    • Increasing amounts of disk space
    • Use of databases
    • Shell access to the server
    • eCommerce facilities
    • Streaming audio / video
    • Adult content!

Choosing A Hosting Provider

  • Choice of hosting provider for simple Web sites that are only static HTML depends on:
    • Cost
    • Reliability
    • Technical support
    • Customer service
  • Choice of hosting provider for complex sites depends on more factors, for example:
    • Operating system desired
    • Scripting language desired
    • Database desired

Choosing A Hosting Provider

  • Many lists of hosting companies, e.g. PCMag
  • Often better to rely on recommendation
  • Recommend DreamHost for UNIX hosting of non-critical sites
  • Used by this site
  • Use Melbourne currently for critical sites

Accessing Hosting Provider

  • Most common method of accessing hosts is FTP (File Transfer Protocol)
  • FTP is inherently insecure because passwords are transmitted in cleartext and can be sniffed by someone else on the network
  • If security is a priority then should access host securely using, for example, SFTP

Common Growth Problems

  • Small Web site with one author is no problem
  • If site is larger or more than one author then many potential problems
  • If site is large then altering the user interface might require editing hundreds of pages
  • Could do global search and replace with text editor, but better to separate user interface from content using SSIs (server-side includes) or equivalent
  • If make changes to live site then users might see mistakes made during editing
  • Therefore have test site where changes are made and then reviewed (ideally by other members of the team) before being uploaded to main server

Common Growth Problems

  • If have multiple authors then is a chance that one author's changes may overwrite another's
  • Need version control where file is locked whilst it is changed
  • Also allows rollback if an error was made
  • Have audit trail
=> content management system

Audience Growth Problems

  • As number of users increases, the upper limit of the software / hardware used might be reached
  • As load increases site may be unusable for all users (not just some)
  • Two general solutions to load problems:
    • Write more efficient software (e.g. optimise database queries or generate static HTML pages from dynamic ones overnight)
    • Get better hardware / infrastructure (e.g. multiple, load-balanced servers)
  • Both have costs (salary or hardware) so neither right or wrong
  • Scale up (vertically) or scale out (horizontally)
  • Many sites have systems in place to deal with traffic bursts

Not Discussed

  • Taking control of an existing site
  • Moving site to a new hosting provider
