Why Version Control?

  • If you end up with files or directories like...
    • logo.png.bak
    • logo.png.bak2
    • index.html.2018-01-03
    • vdwp7.latest
    • vdwp7.2018-02-17b
  • Or you make changes and want to revert to the original, but don't remember what you changed...
  • Or you want multiple people to edit the same set of files...
  • Then you need version control

Advantages Of Version Control

  • Restore: You can revert back to the previous state of a file (or files) at any point in time
  • Undo: Throw away changes and revert to last known good version of file
  • History: Add comments to each change saying why the change was made
  • Ownership: Changes are associated with people (who can be blamed or credited)
  • Synchronisation: Multiple people can edit same files and stay up to date
  • Branching and merging: Make changes to separate copy of files and merge later

Concepts

  • Repository: The database of files (including all versions)
  • Revision: A version of a file in the repository
  • Working Copy: The set of files that one person works on
  • Add: Putting a new file into the repository so it's tracked and under version control
  • Checkout: Getting a file from the repository
  • Checkin or Commit: Adding a new revision of a file to the repository
  • Commit Message: A description of a set of changes

More Concepts

  • Update or Fetch: Get the latest revisions from another repository
  • Revert: Discard changes and replace with the latest revision from the repository
  • Diff: The changes between two revisions of a file
  • Master or Trunk: The primary baseline set of files in the repository
  • Branch: Making a copy of all the files for editing
  • Merge: Merging two sets of changes together
  • Conflict: When changes can't be merged automatically

Typical Scenario

  • Ian adds index.html to the repository
  • Kieren checks out the repository, changes the h1 tag of index.html and commits the new revision with a commit message of "New title"
  • Ian updates his working copy from the repository and sees Kieren has made a change
  • Ian does a diff to see what Kieren changed
  • Ian makes a new branch
  • Ian commits lots of changes to index.html in his branch over the next week
  • Kieren commits a change to index.html during that week
  • Ian merges his branch into the master and resolves a conflict between his changes and Kieren's change

History

  • Local version control systems (e.g. rcs)
  • Centralised version control systems (e.g. cvs or svn)
  • Distributed version control systems (e.g. git)
  • Git created in 2005 by Linus Torvalds (creator of Linux)
  • Git now the most commonly used version control system for Web development

Hosting

  • Centralised hosting repositories (e.g. GitHub or Bitbucket) - give you nice tools and backups
  • GitHub is largest code hoster (67 million repositories & 24 million developers)
  • GitHub offers unlimited open repositories but private repositories require payment
  • Bitbucket offers unlimited free private repositories
