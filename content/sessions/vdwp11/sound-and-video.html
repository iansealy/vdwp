Adding Sound and Video

Sound and video can be effective, however they must be used with care.

Site owners need to consider:

  • Primary purpose/visitor
  • Accessibility
  • Copyright
  • Bandwidth/file size vs Quality trade off

Adding Sound

Sound is commonly distributed in one of the following formats:

  • Wav
  • Real Audio
  • Midi - uses built in "voices" from your sound card
  • MP3 - arguably the most common audio format
  • FLAC - Free Lossless Audio Codec
  • WMA - Windows Media Audio
  • Ogg Vorbis - free, open, and unpatented

See:

  • Basic audio editing

Adding sound 2

How to add sound to your pages:

  1. Downloadable sound clips
  2. Embedded sound clips
  3. Streaming Audio (covered in streaming video)

The visitor will need relevant software or a plug-in to be able to listen to the file.

Downloadable Sound Clips

HTML code for including a sound file as a downloadable link:

<a href="example.mp3">30 seconds of Ian singing, 500K MP3 file</a>

  • Interoperability - a standard link so it will always work
  • Unobtrusive - visitor isn't forced to download the file
  • Flexibility - you can provide several different file types
  • Description - easy to give a description of the file and file size

Embedded Sound Clips

Embedding using the <embed> tag

Invalid element in HTML4/XHTML but formally introduced in HTML5

HTML5 example

If using HTML5, much better to use <audio>.

HTML5 <audio>

In HTML5 audio is natively supported using the <audio> element.

<audio src="example.ogg" controls></audio>

Example

The mark up can also support multiple formats:

<audio>
<source src="example.mp3" type='audio/mpeg; codecs="mp3"'>
<source src="example.oga" type='audio/ogg; codecs="vorbis"'>
<p>Your browser doesn't support the <code>audio</code> element.</p>
</audio>

Good support in current browsers

The browser defines how the 'player' appears.

Video delivery

You have two choices for making video available on the web:

  1. Progressive Streaming (HTTP - Hyper Text Transfer Protocol)
  2. Real Time Streaming (RTSP - Real Time Streaming Protocol)

To work with low bandwidth connections the video will lose most of its clarity and be highly compressed.

Progressive Streaming

Normal download, not true streaming. The video is downloaded to the viewer's computer but may begin playing before download is complete.

It is preferable where bandwidth is likely to be low but this can be problematic for long clips due to their file size.

Common formats are:-

  • MPEG
  • MP4
  • Ogg

Progressive Streaming

Two choices:

  1. Embedded
  2. Downloadable

The visitor will need relevant software or a plug-in to be able to view the file and you should also provide a link to a source of the relevant software/plug-in.

Real Time Streaming

Real Time Streaming Protocol (RTSP) better for long clips and on LAN or broadband/high speed environments.

Requires a streaming server (additional expense/expertise).

  • Real Networks offers a free version of its Helix server
  • Some ISPs offer a streaming server with their Web space
  • Use a service like YouTube or Vimeo

Real Time Streaming 2

The benefits:-

  • Don't need to download whole clip before playing
  • The video resides on the server (can't be downloaded easily)
  • The visitor can jump to any part of the video without having to download the preceding footage
  • Tries to keep pace with user's connection speed, server tries to maintain a constant connection
  • Video for streaming does need careful preparation
  • Video streaming can require greater processing power on the client machine and a good broadband connection

HTML5 <video> element

  • HTML4/XHTML has no standards compliant way of embedding video. Virtually all video uses a plugin.
  • HTML5 provides the <video> element as a standard way to embed video.
  • No restrictions on file format or audio/video codecs:
    • video - H.264, Ogg Theora, VP8
    • audio - MP3, AAC, Ogg Vorbis
  • One <video> element links to multiple file formats.
  • Browsers play the first file they can.
  • No single format/codec combination will be played by all browsers.
  • Browser support is very good.

HTML5 players

  • Flow player
  • JW Player

HTML5 <video> element 2

Example

Code example:

<video width="640" height="480" controls>
<source src="example.mp4" type='video/mp4; codecs="avc1.42E01E, mp4a.40.2"'>
<source src="example.ogg" type='video/ogg; codecs="theora, vorbis"'>
</video>

YouTube uses H.264 HTML5 video.

Accessibility

There are many accessibility issues associated with providing audio and video online. Some of the issues to consider include:

  • Must provide a text alternative for all non-text content
  • Providing captions (subtitles for video)
    • YouTube captions
    • YouTube automatic captions
  • Which assistive technologies support the player you've chosen?
  • Does the player have keyboard controls or is it mouse only?

In general, no single solution will be completely accessible so providing the same information in a variety of forms is recommended.
