Introduction

  • HTML has rules (e.g. <tr> must appear within <table>)
  • If rules are broken then HTML is not valid
  • Rules are known as Document Type Definition (DTD)
    • DOCTYPE specifies DTD and must be the first line in an HTML document
    • Specifies HTML version used, for example HTML 4.01
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

All Documents Must Have A DOCTYPE

XHTML DOCTYPE:
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/transitional.dtd">

HTML 4 and XHTML have several different versions of the DOCTYPE including 'strict' or 'transitional/loose'. For now, use 'transitional/loose'

HTML5 DOCTYPE:
<!DOCTYPE html>

It is important that you use the correct DOCTYPE for the version of HTML you are using.

For new sites we suggest you use HTML5.

Invalid HTML

  • Invalid HTML is:-
    • Not accessible
    • Hard to debug
    • The single biggest reason for display problems when using CSS positioning
  • All your pages should be valid
  • CSS also has rules and must also be valid

Validation

  • W3C HTML Validation Service
  • W3C requires DOCTYPE and an encoding, for example:
    <meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1"> for HTML4.
    For HTML5 use: <meta charset="UTF-8">
  • Nu validator
  • WDG HTML Validator
  • W3C CSS Validation Service
  • Can be accessed via the Web Developer Toolbar in Firefox or Chrome

Tips for validating

  • Errors cascade - once you have one error it can cause others to be reported in the same page
  • When starting out, always fix the first error in the list first and then re-validate and fix the next
  • The validator reports a line and column number - use this to track down the problem
    • Errors might not always be at this point, but will be close by
  • If you don't understand the error message, use the online help for the validator and, failing that, ask on the mailing list
  • If your pages don't display as intended, validate your code and fix any errors. Don't try and diagnose issues when you have invalid code.
  • Make sure the validator is using the correct version of HTML or CSS when validating your code
