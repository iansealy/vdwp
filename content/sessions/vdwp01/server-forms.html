Server-Side Scripting

  • So far have created static Web pages
  • Server-side scripting allows creating dynamic or interactive pages (e.g. database-driven pages)
  • Example:
    • Vehicle Enquiry Service
    • Content is generated dynamically (from a database), rather than having a static page for every car
  • Different to client-side scripting (e.g. Foreign Travel Advice)
  • Code runs on server not in browser

More Server-Side Scripting

  • Normally (i.e. for a static page) a requested file is sent back to the Web browser directly
  • With server-side scripting the requested file is executed as a program and that program's output is sent back to the Web browser
  • Server-side scripts can be written in any language that will run on the computer hosting the Web server (confusingly the computer hosting the Web server software is also often called a Web server)
  • Introduction to the server side

Languages

Examples of programming languages often used to write server-side scripts:

  • PHP
  • Python
  • Perl
  • Ruby
  • C#
  • Java
  • JavaScript (although also used for client-side programming)

Languages used on most popular sites

Reusing Server-Side Scripts

  • Teaching any of these languages is beyond the scope of this course
  • Many Web developers never write their own scripts
  • Easy to reuse scripts that other people have written

What You Need

  • An area on your Web server where files are executed rather than just sent back directly
  • On the DreamHost server, server-side scripts can be run from any directory below htdocs and scripts just need to have the right file extension (e.g. .pl or .php)
  • Is also possible to use scripts hosted by a third party (but beware of advertising)

Example

Example server-side script:

#!/usr/bin/perl
use CGI qw(param);
print "Content-type: text/html\n\n";
print "<html><h1>Hello ", param('username'), "</h1></html>";

Example

  • Is a Perl script
  • Located at http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl
  • Takes a parameter called "username" and displays the value of that parameter after "Hello"
  • Can pass parameter as part of URL
  • http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl?username=Kieren
  • http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl?username=Ian

Forms

  • Using URLs is one way of sending parameters to a server-side script, but most common way is probably to use forms
  • Forms are HTML that allow user to enter information and send it to the Web server
  • All forms are enclosed in <form> elements
  • The action attribute of form element points to server-side script
  • There are three basic elements within a form - <input>, <select> and <textarea>
  • Other elements are necessary (e.g. <label> for accessibility)

Input Element

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl">
<input type="text" name="username" size="10">
<input type="submit" value="Go">
</form>

More Input Element

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl">
<input type="radio" name="username" value="Ian">Ian's first name
<br />
<input type="radio" name="username" value="Kieren">Kieren's first name
<br />
<input type="radio" name="username" value="Ian Sealy">Ian's full name
<br />
<input type="radio" name="username" value="Kieren Pitts">Kieren's full name
<br />
<input type="submit" value="Go">
</form>

Select Element

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl">
<select name="username">
<option value="Ian">Ian's first name</option>
<option value="Kieren">Kieren's first name</option>
<option value="Ian Sealy">Ian's full name</option>
<option value="Kieren Pitts">Kieren's full name</option>
</select>
<input type="submit" value="Go">
</form>

Textarea Element

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl">
<textarea rows="10" cols="40" name="username"></textarea>
<input type="submit" value="Go">
</form>

Hidden Input Element

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl">
<input type="hidden" name="username" value="Ian">
<input type="submit" value="Go">
</form>

Method Attribute

Two ways of accessing a server-side script. Determined by method attribute of form element.

POST:

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl" method="POST">

GET:

<form action="http://iansealy.{{< shortsitename >}}{{< shortyear >}}.bbkweb.org/example.pl" method="GET">

Which is used depends on server-side script. Can POST an unlimited amount of data. GET is limited.
