What Is JavaScript?

  • A scripting language
  • Traditionally used on the client (i.e. Web browser) rather than the server
  • Allows interactivity without reloading the entire page (unlike server-side scripts)
  • Because there is no connection to the server, JavaScript can be fast and therefore allows instant interactivity (e.g. respond to mouse clicks or movement or typing)
  • Examples:
    • Yosemite Weather Trends
    • Ball Pool
    • Here Is Today

Aside: Java And JavaScript

  • JavaScript is a scripting language used on the Web
  • Java is a programming language used for a wide range of applications (which are not necessarily Web-related)
  • Java and JavaScript are completely different

History

  • JavaScript was developed by Netscape for Navigator, but was called Mocha and then later LiveScript
  • Was renamed JavaScript for marketing reasons (even though very different to Java)
  • Navigator 2.0 (1996) first supported JavaScript
  • Microsoft created its own version of JavaScript called JScript (Netscape owned the name)

More History

  • IE 3.0 (1996) first supported JScript, but was very different to JavaScript
  • Confusingly IE 3.0+ (until IE 11) also support a scripting language called VBScript, which is not supported by other browsers
  • Netscape, Microsoft and others collaborated with ECMA to create an official JavaScript specification called ECMAScript
  • Browsers have been ECMAScript-compliant since Navigator 4.06 (JavaScript 1.3) and IE 4 (JScript 3.0+)

Where Is JavaScript?

  • Can be embedded in an HTML tag using event attributes (e.g. onclick and onmouseover):
    <a href="#" onclick="window.alert('Hi')">HTML tag</a>
  • Can be embedded in a page using the script tag:
    <script type="text/javascript">
    window.alert('Hi');
    </script>
  • Can be included from an external file using the script tag:
    <script type="text/javascript" src="example.js"></script>

Latter is ideal because separates behaviour from content.

Popup Window

Popup

<a href="window.html" onclick="window.open('window.html', 'popup', 'width=400,height=300'); return false;">Popup</a>

But URL is now duplicated, so hard to maintain

Popup Window

Popup

<a href="window.html" onclick="window.open(this.href, 'popup', 'width=400,height=300'); return false;">Popup</a>

But JavaScript (behaviour) is mixed with content, so still hard to maintain

Need to separate content and behaviour in the same way we separate content and presentation using CSS

Popup Window

Popup

<a href="window.html" class="popup">Popup</a>

window.onload = makePopups;
function makePopups() {
    var links = document.getElementsByTagName('a');
    for (var i = 0; i < links.length; i++) {
        if (links[i].getAttribute('class') == 'popup') {
            links[i].onclick = function() {
                window.open(this.getAttribute('href'), 'popup', 'width=400,height=300');
                return false;
            }
        }
    }
}

Problems With JavaScript

  • Like CSS, JavaScript does not always work the same in all browsers
  • And is a programming language, so harder to learn than HTML or CSS
  • Often overcome by using a JavaScript library, of which the most popular by far is jQuery
  • Allows shorter, clearer syntax than standard JavaScript

Popup Window

Popup

<a href="window.html" class="popup">Popup</a>

jQuery version:

$(document).ready(function() {
    $('a.popup').click(function() {
        window.open($('a.popup').attr('href'), 'popup', 'width=400,height=300');
        return false;
    });
});
